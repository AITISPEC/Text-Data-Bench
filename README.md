# 📊 TextDataBench

Автоматизированный конвейер для бенчмаркинга, очистки, дедупликации и балансировки текстовых датасетов. Поддерживает **13 форматов** с детерминированным роутингом и стандартизированным выводом.

## ✨ Ключевые возможности
- **13 движков загрузки**: CSV, TSV, Parquet, Feather/Arrow (IPC), Excel, JSONL, JSON, TXT, MD, XML, HDF5, Pickle, HF Datasets.
- **Авто-стандартизация**: Сложные и вложенные структуры автоматически преобразуются в единую колонку `_tdb_text`.
- **Полный пайплайн**: Фильтрация → MinHash-дедупликация → Стратифицированная балансировка → Сбор метрик.
- **Интерактивное CLI**: Удобное меню с поддержкой пакетной обработки (`[0] ALL`).
- **Наглядные отчёты**: Индивидуальные Markdown-файлы для каждого датасета с дельтой метрик «До/После».
- **Безопасный I/O**: Автоматический фоллбэк в `.parquet` при сохранении форматов, не поддерживающих вложенность.
- **GPU-детекция**: Автоматическое определение NVIDIA GPU / Apple MPS при включённой опции `prefer_gpu`.

## 🚀 Установка

Требуется **Python 3.12+**. Для корректной работы на Windows рекомендуется использовать `conda` (решает проблемы с бинарными зависимостями `datasets`, `h5py` и др.).

Скрипт `force-reinstall.ps1` поддерживает гибкие режимы установки:

```powershell
# Тихий режим (только ядро, без зависимостей)
.\force-reinstall.ps1

# Полная установка (разрешает все зависимости из pyproject.toml)
.\force-reinstall.ps1 -deps

# Подробный вывод + дополнительные флаги pip
.\force-reinstall.ps1 -deps -verbose -extra "--upgrade-strategy eager"
```

## 📖 Использование

Запуск интерактивного меню:
```powershell
python -m text_data_bench
```

Пункты меню:
- `[1] Test` — Запуск тестов (pytest -v -m "not slow")
- `[2] Bench` — Обработка датасетов из `tests/fixtures/` → сохранение в `output/`
- `[3] Force Reinstall` — Интерактивный запуск скрипта переустановки
- `[0] Exit` — Выход

💡 **Как работать:**
1. Поместите ваши файлы в `tests/fixtures/`.
2. Выберите `[2] Bench`.
3. Укажите `[0] ALL` для пакетной обработки или номер конкретного файла.
4. Готовые `.parquet` и отчёты `<имя>.md` появятся в папке `output/`.

## ⚙️ Конфигурация

Параметры пайплайна управляются через `configs/pipeline.yaml`:
```yaml
filters:
  min_length: 10      # Минимальная длина текста
  max_length: 10000   # Максимальная длина
  remove_empty: true  # Удалять пустые строки
dedup:
  exact: true         # Точная дедупликация
  fuzzy: true         # Нечёткая дедупликация (MinHash)
  fuzzy_threshold: 0.85 # Порог нечёткого дублирования
  num_perm: 128       # Количество перестановок для MinHash
balance:
  strategy: "stratified" # Стратифицированная балансировка
  group_col: null     # Колонка для группировки (если нужна)
  seed: 42            # Seed для воспроизводимости
pipeline:
  name: "default"
  prefer_gpu: false   # Детекция GPU при старте
output:
  report_path: "./output/reports"
```

## 📦 Поддерживаемые форматы

| Категория | Расширения | Хендлер |
|-----------|------------|---------|
| Табличные | `.csv`, `.tsv`, `.parquet`, `.feather`, `.arrow`, `.xlsx`, `.xls` | `polars_csv`, `polars_tsv`, `polars_parquet`, `polars_ipc`, `polars_excel` |
| Текстовые / NLP | `.jsonl`, `.json`, `.txt`, `.md`, `.xml` | `polars_jsonl`, `polars_json`, `text_lines`, `xml_simple` |
| Специализированные | `.h5`, `.hdf5`, `.pkl`, `.pickle`, `.hf` | (через `engines/core.py`) |

> **Примечание:** Основной пайплайн использует `engines/core.load_and_standardize()`. Модуль `io/loaders.auto_load()` доступен для кастомных сценариев.

│   └── generate_fixtures.py # Генератор тестовых данных
